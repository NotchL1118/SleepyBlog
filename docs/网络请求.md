# 网络请求与数据获取

本文档总结了项目中进行网络请求和数据获取的两种主要方法。

## 1. Server Actions (首选方法)

项目主要使用 Next.js 的 Server Actions 来处理客户端与服务器之间的数据交互。这是一种现代化、高效且安全的方式，可以直接在服务器上执行函数，而无需手动创建 API 端点。

### 工作流程

1.  **定义 Action**: 在 `src/actions/` 目录下创建文件（例如 `src/actions/article.ts`），并在文件顶部声明 `"use server";`。
2.  **实现函数**: Action 函数通常是异步的，它们会调用数据仓库层（Repository）与数据库进行交互。
3.  **标准化返回**: 使用 `ServerActionBuilder` 工具来包装返回结果，确保客户端收到的数据结构一致（包含 `success`, `data`, `message`, `error` 等字段）。
4.  **客户端调用**: 在客户端组件中（必须是 `"use client";` 组件），直接导入并像调用普通异步函数一样调用 Action。

### 使用方法

#### 服务端 Action 定义 (`src/actions/article.ts`)

```typescript
"use server";

import ArticleRepository from "@/repositories/article-repository";
import ServerActionBuilder from "@/lib/server-action";

// ... 其他导入

/**
 * 获取文章列表
 */
export async function getArticleList(params?: { page?: number; limit?: number }) {
  return ServerActionBuilder.execute(async () => await ArticleRepository.getList(params), {
    successMessage: "获取文章列表成功",
    onError: (error) => console.error("Server Action - 获取文章列表失败:", error),
  });
}
```

#### 客户端组件调用 (`src/app/some-page/page.tsx`)

```tsx
"use client";

import { useEffect, useState } from "react";
import { getArticleList } from "@/actions/article";
import ServerActionBuilder from "@/lib/server-action";
import { Article } from "@/types/article";

export default function ArticlePage() {
  const [articles, setArticles] = useState<Article[]>([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const fetchArticles = async () => {
      setLoading(true);
      setError(null);
      try {
        const result = await getArticleList({ page: 1, limit: 10 });
        if (ServerActionBuilder.isSuccess(result)) {
          setArticles(result.data?.items || []);
        } else {
          setError(result.error || "获取文章列表失败");
        }
      } catch (err) {
        setError("网络请求失败：" + (err as Error).message);
      } finally {
        setLoading(false);
      }
    };

    fetchArticles();
  }, []);

  // ... JSX 渲染部分
}
```

## 2. 简单的 Fetch 封装 (次选方法)

对于一些特殊情况，例如调用第三方 API 或 Webhook，项目提供了一个对原生 `fetch` 的简单封装。

**注意**: 此方法不应用于项目内部的数据获取，应优先使用 Server Actions。

### 文件位置

`src/lib/request.ts`

### 使用方法

该工具提供了 `simpleFetch` 和 `simplePost` 两个方法。

```typescript
import { api } from "@/lib/request";

interface ThirdPartyData {
  id: number;
  name: string;
}

// GET 请求
async function fetchThirdPartyData() {
  try {
    const data = await api.fetch<ThirdPartyData>("https://api.example.com/data");
    console.log(data);
  } catch (error) {
    console.error("Failed to fetch data:", error);
  }
}

// POST 请求
async function postDataToWebhook() {
  try {
    const response = await api.post<{ success: boolean }>("/api/webhook", { message: "Hello World" });
    if (response.success) {
      console.log("Webhook triggered successfully");
    }
  } catch (error) {
    console.error("Failed to post data:", error);
  }
}
```
