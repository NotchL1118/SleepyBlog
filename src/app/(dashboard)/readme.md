博客的后台管理系统

1. 通过此页面可以管理所有文章、网站的所有设置
2. 所有后台独有的TypeScript类型都控制在dashboard内，不要污染外部types文件夹，那么主要服务于主站
3. 后台系统仅供于我使用，所以可以不用在乎SEO、性能问题，可以随意使用客户端渲染
4. 在环境变量中存在ADMIN_PASSWORD 和 ADMIN_USERNAME 用于登录，暂时使用该方式进行登录，后续我会有计划进行升级
5. 环境变量中有MONGODB_DASHBOARD_URL变量，用于网站文章外的设置内容进行存储
6. 尽量将影响范围控制在dashboard文件夹内，不要太影响外部代码
7. 界面要美观、简洁，方便我进行管理网站
8. 我可以通过此页面进行文章增删查改等功能
9. 你可以补充后台系统应该有的一些功能，并进行实现

## 使用说明

1. 访问 `/login` 登录后台管理系统
2. 使用环境变量中配置的 ADMIN_USERNAME 和 ADMIN_PASSWORD
3. 登录后可以：
   - 查看仪表板数据统计
   - 管理文章（增删改查）
   - 批量操作文章
   - 配置网站设置（待完成）

## 技术栈

- **框架**: Next.js 15 (App Router)
- **UI**: Tailwind CSS + Lucide React Icons
- **状态管理**: React Hooks
- **数据库**: MongoDB (双数据库连接)
- **认证**: 简单的环境变量验证
- **类型检查**: TypeScript
